# 通常版について
**DRStandeeShader / タテカンシェーダ** <small>（通常版）</small>のフォルダ構造、および基本的な利用方法についてのマニュアルです。

## インポート後のフォルダ構造
通常版が正常にインポートされている場合、`Assets` フォルダ内に以下のフォルダ構造が生成されます。

```text
Assets/
└── VRElectroCenter/
    └── DRStandeeShader/              <- パッケージのルートフォルダ
        ├── Data/
        │   ├── Material/             <- サンプルマテリアル
        │   │   └── Texture/          <- サンプル用テクスチャ
        │   │       ├── Mask/         <- サンプル用マスク用画像
        │   │       ├── MatCap/       <- サンプル用マットキャップ
        │   │       └── NormalMap/    <- サンプル用ノーマルマップ
        │   ├── Script/
        │   │   ├── CSharp/           <- C#ソースコード
        │   │   └── Udon/             <- Udonプログラム
        │   └── Shader/               <- シェーダー本体
        │       └── Editor/           <- カスタムGUI
        └── Prefab/                   <- サンプルPrefab
```

* **`DRStandeeShader/`**
  * 本パッケージのルートフォルダです。このフォルダ直下に、表示サンプルのPrefabが含まれています。

* **`DRStandeeShader/Data/`**
  * シェーダー本体や、サンプル用のデータ（マテリアル、テクスチャ、スクリプトなど）が格納されています。
  * 通常、このフォルダ内のファイルを直接編集・変更・移動する必要はありません。

* **`DRStandeeShader/Prefab/`**
  * 本シェーダーと付属スクリプトを組み合わせた、サンプルPrefabが格納されています。
  * 各サンプルPrefabの利用方法や機能については、本ページ後半の「サンプルPrefabについて」を参照してください。

## シェーダーの利用方法（初心者向け）

> [!NOTE]
> Unityでのマテリアル作成やシェーダー割り当ての操作に**慣れている方は、このセクションを読み飛ばして**次の「マテリアルの設定項目」へお進みください。

本シェーダーは、基本的にUnityの **Quad** オブジェクトへ適用して使用することを想定しています。

### 1. マテリアルの作成と適用
1. **Project** ウィンドウの空いている場所を右クリックします。
2. メニューから **Create > Material** を選択し、任意の名称（例: `MyStandeeMaterial` 等）でマテリアルを新規作成します。
3. 作成したマテリアルを選択し、**Inspector** ウィンドウ最上部の **Shader** ドロップダウンをクリックします。
4. リストから **VRElectroCenter > DRStandeeShader** を選択します。

### 2. Quadオブジェクトへの適用
1. **Hierarchy** ウィンドウの空いている場所を右クリックし、**3D Object > Quad** を作成します。
2. 設定が完了したマテリアルを、作成した **Quad** へ直接ドラッグ＆ドロップすれば完了です。<br />
   <small>※ イラストのアスペクト比（縦横比）等に合わせて、Quadの `Scale` を調整してください。</small>

## マテリアルの設定項目

本シェーダーを適用したマテリアルの **Inspector** ウィンドウに表示される設定項目の一覧です。

### グローバル設定

* **厚みの調整**
  * 擬似的に描画される厚みの度合いを調整します。<br/>値を大きくするほど厚みが増しますが、処理仕様の関係等により側面部の描画が荒くなる場合があります。

* **ライティング方式**
  * シェーダー全体のライティング方式を選択します。
  * **Unlit**：シーン内のライトの影響を受けず、一定の明るさで表示されます。
  * **Lit**：シーン内のライトの影響を受けて描画されます。

* **最低輝度**
  * **ライティング方式** が **Lit** の場合にのみ表示されます。
  * 暗い場所での「最低限の明るさ」を設定します。値を大きくするほど、影の部分の明るさの下限が引き上げられます。

* **Render Queue**
  * オブジェクトの描画順を設定する、Unity標準の設定項目です。
  * 基本的には初期設定のまま使用してください。値を変更すると、意図した描画結果にならない場合があります。

### 前面描画設定

* **描画ソース**
  * 前面の描画方法を選択します。
  * **Texture**：テクスチャを使用して描画します。
  * **Color**：単色で描画します。

::: details 「Texture」選択時

* **テクスチャ**
  * 前面に表示するテクスチャを設定します。
  * テクスチャのタイリング（Tiling）およびオフセット（Offset）の調整が可能です。
  * **【重要】** Unityにインポートした画像の **Inspector** ウィンドウ（Texture Import Settings）にて、**Alpha is Transparency** のチェックが有効になっている画像を設定してください。

:::

::: details 「Color」選択時

* **カラー**
  * 前面を単色で描画します。<br/>
    <small>※ 仕様上、カラーピッカーでアルファ値（透明度）を設定しても反映はされません。</small>

:::

* **ノーマルマップを使用**
  * 前面にノーマルマップを適用し、凹凸の質感を表現します。

::: details 「ノーマルマップを使用」有効時

* **マップ**
  * 前面に適用するノーマルマップを設定します。
  * テクスチャのタイリング（Tiling）およびオフセット（Offset）の調整が可能です。

* **強度**
  * ノーマルマップによる凹凸の強調具合を調整します。数値を大きくすると陰影が深くなります。

:::

* **マットキャップを使用**
  * 前面にマットキャップを適用します。

::: details 「マットキャップを使用」有効時

* **テクスチャ**
  * 使用するマットキャップテクスチャを設定します。

* **強度**
  * マットキャップによる質感や光沢の反射具合を調整します。数値を大きくするとツヤが強くなります。

:::

* **エミッションを使用**
  * 前面にエミッション（自己発光）を適用します。

::: details 「エミッションを使用」有効時

* **カラー**
  * エミッションの色を設定します。

* **マップ**
  * エミッションとして表示するテクスチャを設定します。
  * テクスチャのタイリング（Tiling）およびオフセット（Offset）の調整が可能です。

* **マスク**
  * エミッションを適用する範囲を制御します。
  * 白い部分が強く発光し、黒い部分は発光しません。
  * テクスチャのタイリング（Tiling）およびオフセット（Offset）の調整が可能です。

* **強度**
  * エミッションの明るさを調整します。数値を大きくすると発光がより強くなります。

:::

### 切り抜き・クロマキー設定

* **切抜きの閾値**
  * オブジェクトを切り抜くかどうかを判定する基準値を設定します。
  * 値を大きくするほど、切り抜かれる範囲が広くなります。

* **切り抜き基準**
  * オブジェクトの輪郭を切り抜くための判定方法を選択します。
  * **MainTex_Alpha**：メインテクスチャのアルファ値（透明度）を使用して切り抜きを行います。<br/>
    <small>※ 前面描画のテクスチャに一般的な透過PNGを使用する場合はこちらを選択してください。</small>
  * **Mask_Texture_Grayscale**：マスク画像の明るさを使用して切り抜きを行います。
  * **ChromaKey_Green**：マスク画像内の緑色に近い部分を切り抜きます。
  * **ChromaKey_Blue**：マスク画像内の青色に近い部分を切り抜きます。

::: details 「Mask_Texture_Grayscale」選択時

* **マスク**
  * 切り抜き判定に使用するマスク画像を設定します。
  * 白い部分が表示され、黒い部分が切り抜かれます。
  * テクスチャのタイリング（Tiling）およびオフセット（Offset）の調整が可能です。

* **マスクを反転**
  * マスク画像の白黒を反転して判定します。

:::

::: details 「ChromaKey_Green」または「ChromaKey_Blue」選択時

* **マスク**
  * クロマキー判定に使用する画像を設定します。
  * テクスチャのタイリング（Tiling）およびオフセット（Offset）の調整が可能です。

:::
### 背面描画設定

* **描画ソース**
  * 背面の描画方法を選択します。
  * **MainTex**：前面に設定したテクスチャを使用して描画します。
  * **BackTex**：背面専用のテクスチャを別途設定して描画します。
  * **Color**：単色で描画します。

* **背面の明るさ補正**
  * 背面の明るさを調整します。
  * **描画ソース**が **MainTex** または **BackTex** の場合に適用されます。

::: details 「MainTex」選択時

前面描画設定のテクスチャを使用して描画します。

:::

::: details 「BackTex」選択時

* **テクスチャ**
  * 背面に表示するテクスチャを設定します。
  * テクスチャのタイリング（Tiling）およびオフセット（Offset）の調整が可能です。

:::

::: details 「Color」選択時

* **カラー**
  * 背面を単色で描画します。<br/>
    <small>※ カラーピッカーでアルファ値（透明度）を設定しても反映されません。</small>

:::

* **ノーマルマップを使用**
  * 背面にノーマルマップを適用し、凹凸の質感を表現します。

::: details 「ノーマルマップを使用」有効時

**描画ソース** が **MainTex** の場合は、前面描画設定のノーマルマップを使用します。

**描画ソース** が **BackTex** または **Color** の場合は、以下の設定が表示されます。

* **マップ**
  * 背面に適用するノーマルマップを設定します。
  * テクスチャのタイリング（Tiling）およびオフセット（Offset）の調整が可能です。

* **強度**
  * ノーマルマップによる凹凸の強調具合を調整します。数値を大きくすると陰影が深くなります。

:::

* **マットキャップを使用**
  * 背面にマットキャップを適用します。

::: details 「マットキャップを使用」有効時

* **テクスチャ**
  * 使用するマットキャップテクスチャを設定します。

* **強度**
  * マットキャップによる質感や光沢の反射具合を調整します。数値を大きくするとツヤが強くなります。

:::

* **エミッションを使用**
  * 背面にエミッション（自己発光）を適用します。

::: details 「エミッションを使用」有効時

**描画ソース** が **MainTex** の場合は、前面描画設定のエミッション設定を使用します。

**描画ソース** が **BackTex** または **Color** の場合は、以下の設定が表示されます。

* **カラー**
  * エミッションの色を設定します。

* **マップ**
  * エミッションとして表示するテクスチャを設定します。
  * テクスチャのタイリング（Tiling）およびオフセット（Offset）の調整が可能です。

* **マスク**
  * エミッションを適用する範囲を制御します。
  * 白い部分が強く発光し、黒い部分は発光しません。
  * テクスチャのタイリング（Tiling）およびオフセット（Offset）の調整が可能です。

* **強度**
  * エミッションの明るさを調整します。数値を大きくすると発光がより強くなります。
