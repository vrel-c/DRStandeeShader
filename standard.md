# 通常版について
**DRStandeeShader / タテカンシェーダ** <small>（通常版）</small>のフォルダ構造、および基本的な利用方法についてのマニュアルです。

## インポート後のフォルダ構造
通常版が正常にインポートされている場合、`Assets` フォルダ内に以下のフォルダ構造が生成されます。

```text
Assets/
└── VRElectroCenter/
    └── DRStandeeShader/              <- パッケージのルートフォルダ
        ├── Data/
        │   ├── Material/             <- サンプルマテリアル
        │   │   └── Texture/          <- サンプル用テクスチャ
        │   │       ├── Mask/         <- サンプル用マスク用画像
        │   │       ├── MatCap/       <- サンプル用マットキャップ
        │   │       └── NormalMap/    <- サンプル用ノーマルマップ
        │   ├── Script/
        │   │   ├── CSharp/           <- C#ソースコード
        │   │   └── Udon/             <- Udonプログラム
        │   └── Shader/               <- シェーダー本体
        │       └── Editor/           <- カスタムGUI
        └── Prefab/                   <- サンプルPrefab
```

* **`DRStandeeShader/`**
  * 本パッケージのルートフォルダです。このフォルダ直下に、表示サンプルのPrefabが含まれています。

* **`DRStandeeShader/Data/`**
  * シェーダー本体や、サンプル用のデータ（マテリアル、テクスチャ、スクリプトなど）が格納されています。
  * 通常、このフォルダ内のファイルを直接編集・変更・移動する必要はありません。

* **`DRStandeeShader/Prefab/`**
  * 本シェーダーと付属スクリプトを組み合わせた、サンプルPrefabが格納されています。
  * 各サンプルPrefabの利用方法や機能については、本ページ後半の「サンプルPrefabの解説」を参照してください。

## シェーダーの利用方法（初心者向け）

> [!NOTE]
> Unityでのマテリアル作成やシェーダー割り当ての操作に**慣れている方は、このセクションを読み飛ばして**次の「マテリアルの設定項目」へお進みください。

本シェーダーは、基本的にUnityの **Quad** オブジェクトへ適用して使用することを想定しています。

### 1. マテリアルの作成と適用
1. **Project** ウィンドウの空いている場所を右クリックします。
2. メニューから **Create > Material** を選択し、任意の名称（例: `MyStandeeMaterial` 等）でマテリアルを新規作成します。
3. 作成したマテリアルを選択し、**Inspector** ウィンドウ最上部の **Shader** ドロップダウンをクリックします。
4. リストから **VRElectroCenter > DRStandeeShader** を選択します。

### 2. Quadオブジェクトへの適用
1. **Hierarchy** ウィンドウの空いている場所を右クリックし、**3D Object > Quad** を作成します。
2. 設定が完了したマテリアルを、作成した **Quad** へ直接ドラッグ＆ドロップすれば完了です。<br />
   <small>※ イラストのアスペクト比（縦横比）等に合わせて、Quadの `Scale` を調整してください。</small>

## マテリアルの設定項目

本シェーダーを適用したマテリアルの **Inspector** ウィンドウに表示される設定項目の一覧です。

### グローバル設定

* **厚みの調整**
  * 擬似的に描画される厚みの度合いを調整します。<br/>値を大きくするほど厚みが増しますが、処理仕様の関係等により側面部の描画が荒くなる場合があります。

* **ライティング方式**
  * シェーダー全体のライティング方式を選択します。
  * **Unlit**：シーン内のライトの影響を受けず、一定の明るさで表示されます。
  * **Lit**：シーン内のライトの影響を受けて描画されます。

* **最低輝度**
  * **ライティング方式** が **Lit** の場合にのみ表示されます。
  * 暗い場所での「最低限の明るさ」を設定します。値を大きくするほど、影の部分の明るさの下限が引き上げられます。

* **Render Queue**
  * オブジェクトの描画順を設定する、Unity標準の設定項目です。
  * 基本的には初期設定のまま使用してください。値を変更すると、意図した描画結果にならない場合があります。

### 前面描画設定

* **描画ソース**
  * 前面の描画方法を選択します。
  * **Texture**：テクスチャを使用して描画します。
  * **Color**：単色で描画します。

::: details 「Texture」選択時

* **テクスチャ**
  * 前面に表示するテクスチャを設定します。
  * テクスチャのタイリング（Tiling）およびオフセット（Offset）の調整が可能です。
  * **【重要】** Unityにインポートした画像の **Inspector** ウィンドウ（Texture Import Settings）にて、**Alpha is Transparency** のチェックが有効になっている画像を設定してください。

:::

::: details 「Color」選択時

* **カラー**
  * 前面を単色で描画します。<br/>
    <small>※ 仕様上、カラーピッカーでアルファ値（透明度）を設定しても反映はされません。</small>

:::

* **ノーマルマップを使用**
  * 前面にノーマルマップを適用し、凹凸の質感を表現します。

::: details 「ノーマルマップを使用」有効時

* **マップ**
  * 前面に適用するノーマルマップを設定します。
  * テクスチャのタイリング（Tiling）およびオフセット（Offset）の調整が可能です。

* **強度**
  * ノーマルマップによる凹凸の強調具合を調整します。数値を大きくすると陰影が深くなります。

:::

* **マットキャップを使用**
  * 前面にマットキャップを適用します。

::: details 「マットキャップを使用」有効時

* **テクスチャ**
  * 使用するマットキャップテクスチャを設定します。

* **強度**
  * マットキャップによる質感や光沢の反射具合を調整します。数値を大きくするとツヤが強くなります。

:::

* **エミッションを使用**
  * 前面にエミッション（自己発光）を適用します。

::: details 「エミッションを使用」有効時

* **カラー**
  * エミッションの色を設定します。

* **マップ**
  * エミッションとして表示するテクスチャを設定します。
  * テクスチャのタイリング（Tiling）およびオフセット（Offset）の調整が可能です。

* **マスク**
  * エミッションを適用する範囲を制御します。
  * 白い部分が強く発光し、黒い部分は発光しません。
  * テクスチャのタイリング（Tiling）およびオフセット（Offset）の調整が可能です。

* **強度**
  * エミッションの明るさを調整します。数値を大きくすると発光がより強くなります。

:::

### 切り抜き・クロマキー設定

* **切り抜きの閾値**
  * オブジェクトを切り抜くかどうかを判定する基準値を設定します。
  * 値を大きくするほど、切り抜かれる範囲が広くなります。

* **切り抜き基準**
  * オブジェクトの輪郭を切り抜くための判定方法を選択します。
  * **MainTex_Alpha**：メインテクスチャのアルファ値（透明度）を使用して切り抜きを行います。<br/>
    <small>※ 前面のテクスチャに一般的な透過PNGを使用する場合、基本的にはこちらを選択してください。</small>
  * **Mask_Texture_Grayscale**：別途設定したマスク画像の明るさを使用して切り抜きを行います。
  * **ChromaKey_Green**：別途設定したマスク画像内の緑色に近い部分を切り抜きます。
  * **ChromaKey_Blue**：別途設定したマスク画像内の青色に近い部分を切り抜きます。

::: details 「Mask_Texture_Grayscale」選択時

* **マスク**
  * 切り抜き判定に使用するマスク画像を設定します。
  * 白い部分が表示され、黒い部分が切り抜かれます。
  * テクスチャのタイリング（Tiling）およびオフセット（Offset）の調整が可能です。

* **マスクを反転**
  * マスク画像の白黒を反転して判定します。

:::

::: details 「ChromaKey_Green」または「ChromaKey_Blue」選択時

* **マスク**
  * クロマキー判定に使用する画像を設定します。
  * テクスチャのタイリング（Tiling）およびオフセット（Offset）の調整が可能です。

:::
### 背面描画設定

* **描画ソース**
  * 背面の描画方法を選択します。
  * **MainTex**：前面に設定したテクスチャを使用して描画します。
  * **BackTex**：背面専用のテクスチャを別途設定して描画します。
  * **Color**：単色で描画します。

* **背面の明るさ補正**
  * 背面の明るさを調整します。
  * **描画ソース**が **MainTex** または **BackTex** の場合に適用されます。

::: details 「MainTex」選択時

前面描画設定のテクスチャを使用して描画します。<br/>
<small>※ 独自設定項目はありません。</small>

:::

::: details 「BackTex」選択時

* **テクスチャ**
  * 背面に表示するテクスチャを設定します。
  * テクスチャのタイリング（Tiling）およびオフセット（Offset）の調整が可能です。

:::

::: details 「Color」選択時

* **カラー**
  * 背面を単色で描画します。<br/>
    <small>※ カラーピッカーでアルファ値（透明度）を設定しても反映されません。</small>

:::

* **ノーマルマップを使用**
  * 背面にノーマルマップを適用し、凹凸の質感を表現します。

::: details 「ノーマルマップを使用」有効時

**描画ソース** が **MainTex** の場合は、前面描画設定のノーマルマップを使用します。

**描画ソース** が **BackTex** または **Color** の場合は、以下の設定が表示されます。

* **マップ**
  * 背面に適用するノーマルマップを設定します。
  * テクスチャのタイリング（Tiling）およびオフセット（Offset）の調整が可能です。

* **強度**
  * ノーマルマップによる凹凸の強調具合を調整します。数値を大きくすると陰影が深くなります。

:::

* **マットキャップを使用**
  * 背面にマットキャップを適用します。

::: details 「マットキャップを使用」有効時

* **テクスチャ**
  * 使用するマットキャップテクスチャを設定します。

* **強度**
  * マットキャップによる質感や光沢の反射具合を調整します。数値を大きくするとツヤが強くなります。

:::

* **エミッションを使用**
  * 背面にエミッション（自己発光）を適用します。

::: details 「エミッションを使用」有効時

**描画ソース** が **MainTex** の場合は、前面描画設定のエミッション設定を使用します。

**描画ソース** が **BackTex** または **Color** の場合は、以下の設定が表示されます。

* **カラー**
  * エミッションの色を設定します。

* **マップ**
  * エミッションとして表示するテクスチャを設定します。
  * テクスチャのタイリング（Tiling）およびオフセット（Offset）の調整が可能です。

* **マスク**
  * エミッションを適用する範囲を制御します。
  * 白い部分が強く発光し、黒い部分は発光しません。
  * テクスチャのタイリング（Tiling）およびオフセット（Offset）の調整が可能です。

* **強度**
  * エミッションの明るさを調整します。数値を大きくすると発光がより強くなります。

:::

### 側面設定

* **彩色タイプ**
  * 側面（厚み部分）の描画方法を選択します。
  * **Texture**：前面描画設定のテクスチャを使用して描画します。
  * **Color**：2色を指定して描画します。

::: details 「Texture」選択時

前面描画設定のテクスチャを使用して側面を描画します。

* **側面の明るさ補正**
  * 側面に表示されるテクスチャの明るさを調整します。

:::

::: details 「Color」選択時

側面カラーは、「前」と「後」の2色で滑らかにグラデーションする形で描画されます。

* **側面カラー（前）**
  * 側面の前方（正面側）の色を設定します。

* **側面カラー（後）**
  * 側面の後方（背面側）の色を設定します。

:::

* **側面発光色**
  * 側面に適用するエミッション（自己発光）の色を設定します。<br/>
    <small>※ エミッションの必要がない場合、黒色（初期値）を指定してください。</small>

* **強度**
  * 側面のエミッションの明るさを調整します。数値を大きくすると発光がより強くなります。

### 高度な設定 (ADVANCED)

> [!TIP]
> 当シェーダーは、 **レイマーチング**という描画手法を利用して1枚のQuadから立体的な厚みを擬似的に表現する処理を行っています。
> 
> このセクションでは、その描画品質や処理負荷、遠距離描画時の品質調整（LOD）など、内部的な動作に関するパラメータを変更することができます。<br/>ただし、基本的には既定値のまま使用してください。描画品質の最適化や特殊な用途など、特定の目的・知識がある場合を除き、設定の変更は非推奨です。

::: details 詳細

* **Max Thickness (m)**
  * `Depth Strength` が最大値（1.0）のときに適用される厚みの上限をメートル単位で設定します。
  * 通常は変更する必要はありません。

* **Min Steps**
  * `Depth Strength` が小さい場合のレイマーチング回数を設定します。
  * 値を大きくすると描画品質が向上する場合がありますが、処理負荷も増加します。

* **Max Steps**
  * `Depth Strength` が大きい場合のレイマーチング回数を設定します。
  * 値を大きくすると厚みが大きい場合の描画品質が向上する場合がありますが、処理負荷も増加します。

* **LOD Start Dist**
  * レイマーチング回数の削減を開始する距離を設定します。
  * この距離を超えると、距離に応じてレイマーチング回数が徐々に減少します。

* **LOD End Dist**
  * レイマーチング回数の削減が完了する距離を設定します。
  * この距離以降は **LOD Minimum Steps** の値で描画されます。

* **LOD Minimum Steps**
  * 遠距離描画時に使用される最小レイマーチング回数を設定します。
  * 値を小さくすると処理負荷を軽減できますが、描画品質が低下する場合があります。

* **Shadow Fade Start (Angle 0-90)**
  * オブジェクトを斜め方向から見た際に、影が徐々に薄くなり始める角度を設定します。
  * 通常は規定値のまま使用してください。

* **Shadow Full Hide (Angle 0-90)**
  * オブジェクトを真横方向から見た際に、影が完全に描画しなくなる角度を設定します。
  * 通常は規定値のまま使用してください。

* **Enable GPU Instancing**
  * Unity標準の設定です。
  * 同一マテリアルを使用するオブジェクトを効率よく描画できる場合があります。

* **Double Sided Global Illumination**
  * Unity標準の設定です。
  * ライトベイク時に両面へグローバルイルミネーションを適用するかを設定します。

:::

## サンプルPrefabの解説

通常版には、実際の利用シーンを想定したサンプルPrefabがいくつか同梱されています。

これらはシェーダー単体の挙動確認用として利用できるほか、簡易的なスクリプト（Udon / U#）による挙動制御を含む構成のものもあります。<br/>
スクリプトについては、他のオブジェクトへコンポーネントとして追加して利用することも可能です。

::: details `DRStandeeShader/Prefab/Sample_FaceCutoutBoard`

### 顔出し看板

ボードの一部を切り抜き、プレイヤーが顔を出す用途を想定したサンプルです。<br/>
切り抜きマスクや厚み表現の基本動作のサンプルとしてご利用ください。

:::

### 画像が切り替わる看板（GraphicDresser）

* `DRStandeeShader/Prefab/Sample_Standee_GraphicDresser`

このサンプルには、**看板に設定されたマテリアルのテクスチャを、複数枚の画像から順番に切り替える機能**を持つ独自のUdonSharpスクリプトが付与されています。

クリック（Interact）操作により、表示画像が順番に切り替わります。<br/>
また、VRChat内での同期に対応しており、同一ワールド内の他プレイヤーにも同じ表示状態が反映されます。

#### 動作仕様
* クリックするたびに次のテクスチャへ切り替え
* テクスチャはインスペクタで配列登録
* 指定マテリアルの _MainTex を直接更新
* インデックス値をUdon同期で共有

#### 補足
* 本スクリプトはマテリアルの内容を直接変更するため、同一マテリアルを共有している他のオブジェクトにも影響します。
* ネットワーク同期はインデックス値のみを共有し、実際のテクスチャは各クライアント側で適用されます。

---

### マテリアルが切り替わる看板（MaterialDresser）

* `DRStandeeShader/Prefab/Sample_Standee_MaterialDresser`

このサンプルには、**MeshRendererに割り当てられたマテリアルを、複数のプリセットから順番に切り替える機能**を持つ独自のUdonSharpスクリプトが付与されています。

クリック（Interact）操作により、表示マテリアルが順番に切り替わります。<br/>
また、VRChat内の同期に対応しており、同一ワールド内の他プレイヤーにも同じ状態が反映されます。

#### 動作仕様
* クリックごとに次のマテリアルへ切り替え
* マテリアル配列はインスペクタで設定
* 指定メッシュレンダラーの material を直接変更
* インデックスをUdon同期で共有

#### 補足
* マテリアルを直接差し替えるため、同じマテリアルを複数オブジェクトで共有している場合でも影響は独立します。
* ネットワーク同期はインデックス値のみを共有し、実際のマテリアルは各クライアント側で適用されます。

---

### 常にプレイヤーの方向を向く看板（TraceHeadingHandler）
* `DRStandeeShader/Prefab/Sample_Billboard`

このサンプルには、**オブジェクトの向きをプレイヤーや任意対象へ追従させる機能**を持つ独自のUdonSharpスクリプトが付与されています。

単純なビルボード処理に加え、特定プレイヤー・オブジェクト・座標など複数の追従モードを切り替えることができます。

#### 動作仕様
* ローカルプレイヤー／指定プレイヤー／オブジェクト／座標への追従に対応
* 頭・腰・足など追従基準点の選択が可能
* 垂直方向（Pitch）回転の有無、180度反転の制御が可能
* LateUpdateのタイミングで毎フレーム回転更新

#### 補足
* 選択する追従モード（プレイヤー、オブジェクト、座標）によってターゲットの指定方法や更新挙動が異なるため、用途に合わせて各プロパティを調整してください。
